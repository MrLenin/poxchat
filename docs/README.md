# PoxChat Documentation

Organized by [PARA](https://fortelabs.com/blog/para/):

- **`projects/`** — active efforts with a goal/endpoint. Each file is a single
  initiative being actively worked. Move to `archives/` when the goal is met
  or the work is abandoned.
- **`areas/`** — ongoing responsibilities with no end date (e.g. release
  process, protocol compatibility tracking, build-system upkeep).
- **`resources/`** — reference material that isn't tied to a specific project
  (protocol notes, design rationale, links worth keeping).
- **`archives/`** — inactive items moved from the other three buckets. Kept
  for historical context, not for ongoing work.

A file's bucket is determined by whether work is currently being done on it,
not by topic. The same subject can move between `projects/` and `archives/`
as work starts and stops.
